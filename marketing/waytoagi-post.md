# waytoAGI 社区帖子

**标题：** 我让 AI 生成了一堆 HTML，换个设备就看不了了——所以我做了这个

---

上周用 Claude 帮我做了一个数据报告，生成的 HTML 还挺好看的，有图表有交互。

然后我想在手机上看一眼。

打不开。

本地 HTML 文件，微信里没法直接打开，Safari 也不行。我把文件发给自己，还是打不开。最后只能截个图发朋友圈——一个有交互的数据报告，变成了一张死图。

后来我想把一个 agent 生成的产品页发给客户看，同样的问题。HTML 文件躺在我电脑上，客户看不到，就是个废物。

我在各种工具里翻了一圈，没找到一个好用的。要么得注册，要么得配置域名，要么得搞一堆部署流程。我只是想让一个 HTML 页面能被别人打开，怎么就这么难？

后来我自己做了一个。

逻辑很简单：一个 API 调用进去，一个公网 URL 出来。

```
curl -X POST https://shippage.ai/v1/publish \
  -H "Content-Type: application/json" \
  -d '{"html": "<h1>Hello</h1>", "title": "My Page"}'
```

返回一个 URL，手机上直接打开，微信里直接打开，发给任何人都能看。

最关键的是：不需要注册。第一次调用会自动帮你创建一个 agent 身份，凭证存在本地，之后就自动带上了。人类用户完全不需要出现在注册流程里。

我给 OpenClaw 做了一个 skill，装了之后对 agent 说"发布这个页面"就行了：

```
clawhub install shippage
```

技术上没什么花活。整个后端就是一个 Cloudflare Worker，HTML 存 R2，元数据存 KV。成本几乎为零——一万个用户大概 $5/月。

免费用户每月 20 次发布，14 天保留，500KB 上限。够用了。

这个东西对你有用吗？你平时 agent 生成的内容是怎么分享给别人的？还缺什么功能的话告诉我。
